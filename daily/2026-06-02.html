Google MCP Toolbox DNS重绑定致企业数据库裸奔：AI Agent基础设施首现高危漏洞

CVE-2026-9739 CVSS 9.4，SSE传输路径硬编码 CORS 通配符，无需认证即可窃取企业数据库

1 严重

CVE-2026-9739 — Google MCP Toolbox DNS重绑定致企业数据库裸奔

CVSS 4.0 = 9.4 AI基础设施 受影响：Google MCP Toolbox for Databases（SSE 传输模式）

MCP Toolbox 是 Google 推出的数据库桥接工具，允许 AI Agent 通过 MCP 协议访问企业数据库。其 SSE 传输路径中硬编码了 Access-Control-Allow-Origin: *，使得 DNS 重绑定攻击可绕过浏览器同源策略。

攻击流程：

攻击者诱导用户访问恶意网站 恶意脚本触发DNS重绑定 浏览器域名解析从外部IP切换到内网MCP Toolbox SSE连接建立，Origin校验被绕过 攻击者通过浏览器向数据库注入命令/提取数据

值得关注的原因：

  • AI Agent 基础设施首现高危漏洞：MCP 协议正快速成为 AI Agent 连接外部工具的标准
  • 攻击门槛极低：无需认证，仅需诱导用户访问恶意页面即可触发
  • 影响面广泛：所有使用 SSE 传输模式且未做网络隔离的实例均受影响

防御建议：审查所有 MCP Toolbox 实例，收紧 Origin 校验，最小权限配置数据库凭证，网络隔离避免公网暴露。

2 严重

Gogs 零日 RCE — 77天无补丁，2400+实例裸奔

CVSS = 9.4 零日无补丁 受影响：Gogs 0.14.2、0.15.0+dev

Rapid7 研究员 Jonah Burgess 于3月17日报告 Gogs 参数注入导致 RCE 的漏洞，维护者3月28日确认收到后再无动作，至今未发布补丁，已持续77天。

利用链：

攻击者注册Gogs账户 创建仓库启用Rebase Merging 提交含恶意分支名的PR 触发Rebase，分支名注入--exec参数 任意命令执行 读取所有仓库/转储凭证

值得关注的原因：

  • 77天无响应：维护者确认收到后再无动作，至今未发布补丁
  • 利用门槛极低：默认配置下只需注册账户即可完成攻击
  • 暴露面：互联网上超过 2,400 个 Gogs 实例暴露
  • 影响严重：可读取所有仓库、转储密码哈希/API Token/SSH密钥/2FA 秘密

Carnival 邮轮数据泄露 — 近600万旅客护照信息外泄

影响人数：约 600 万 攻击方式：社会工程学

Carnival Corporation 于6月1日正式披露，2026年4月发生的数据泄露事件导致近600万客户的敏感信息被未授权访问。

泄露数据类型：

数据类型敏感级别
姓名、地址基本身份信息
电子邮件、电话联系方式
出生日期个人标识
护照号码🔴 高敏感
驾照号码🔴 高敏感

值得关注的原因：

  • 护照号码泄露：属于最高敏感级别的身份文件信息，可用于身份盗窃、伪造旅行证件
  • 攻击方式：社会工程学欺骗员工获取系统访问权限
  • 影响规模：600万用户量级在旅游行业数据泄露中属于重大事件

Oracle CVE-2026-21992 — Identity Manager 未认证RCE（CVSS 9.8）

CVSS = 9.8 IAM基础设施 Oracle 6月1日发布额外安全警报

未认证攻击者可通过 HTTP 网络访问直接利用此漏洞，成功利用可完全接管 Oracle Identity Manager 和 Web Services Manager。

值得关注的原因：

  • CVSS 9.8 满分级：未认证 + 网络可达 + 完全接管
  • IAM基础设施沦陷：Identity Manager 是企业身份认证核心，被接管意味着攻击者可伪造任意身份
  • 6月1日额外警报：Oracle 发布额外安全警报通常意味着在野利用或威胁升级
  • 历史模式：Oracle IAM 组件曾多次成为国家级APT的目标
